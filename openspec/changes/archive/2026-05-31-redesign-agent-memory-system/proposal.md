## Why

当前 `agent_service` 的记忆系统只是基于会话 ID 的进程内短窗口消息列表，服务重启后丢失，不能在团队、租户或集群范围共享，也缺少长期记忆的显式开关、删除和审计能力。随着 AI 运维助手承担更多排障、报告和知识辅助工作，需要把“短期对话上下文”和“可治理的团队级长期记忆”拆分清楚，避免上下文丢失、记忆污染和不可解释的自动写入。

## What Changes

- 新增团队、租户、集群共享的长期记忆能力，用于保存稳定的运维偏好、环境约定、集群知识、已确认事故结论和常见处置流程。
- 新增短期会话记忆与会话摘要能力，替代当前只保留最近 6 条消息的简单窗口策略。
- 新增长期记忆显式开关，支持全局、团队、租户或集群级别控制长期记忆是否写入和参与召回。
- 新增长期记忆查看、删除和按作用域清空能力，删除时需要同步清理持久化记录和语义检索索引。
- 新增记忆治理字段，包括来源、作用域、置信度、创建方式、人工确认状态、创建时间、最后使用时间和过期时间。
- 新增 `agent_service` 自有持久化存储访问，用于保存记忆数据；监控事实仍然只能通过 `api_gateway` 查询。
- 调整聊天 prompt 组装流程，按固定指令、会话摘要、最近消息、长期记忆召回、知识库文档和实时监控事实的顺序组织上下文。

## Capabilities

### New Capabilities

- `agent-memory`: 定义 `agent_service` 的短期会话记忆、会话摘要、长期共享记忆、记忆开关、删除治理和 prompt 注入行为。

### Modified Capabilities

当前没有已存在的 OpenSpec capability 需要修改。

## Impact

- 影响 `agent_service/internal/session/memory`：需要从简单进程内窗口升级为可扩展的记忆管理抽象。
- 影响 `agent_service/internal/handler/agent`：聊天和流式聊天需要通过统一记忆管理器加载、写入和治理记忆。
- 影响 `agent_service/internal/ai/pipeline/chat`：prompt 输入需要支持会话摘要、长期记忆召回结果和最近消息。
- 影响 `agent_service/internal/storage`：需要新增 agent 自有持久化记忆存储，并与语义检索索引保持一致。
- 影响配置：需要新增长期记忆启停、召回数量、摘要窗口、存储连接和保留策略等配置项。
- 影响 API：需要新增记忆查看、开关、删除或清空接口。
- 影响测试：需要覆盖作用域隔离、开关行为、删除一致性、摘要更新、长期记忆召回和聊天流程集成。
